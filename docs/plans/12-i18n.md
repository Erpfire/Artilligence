# Internationalization (i18n)

## Scope
- **Member-facing pages**: English + Hindi
- **Admin panel**: English only
- **Product data**: Bilingual (name and description in both languages)

## Library
**next-intl** — lightweight, works well with Next.js App Router

## Implementation

### Language Detection
1. Check user's `preferred_language` from profile (if logged in)
2. Default to English for guests and admin
3. Language toggle in member nav: `EN | हिं`

### Translation Files

```
src/i18n/
├── en.json    # English translations
└── hi.json    # Hindi translations
```

## Number & Currency Formatting
- Use `Intl.NumberFormat('en-IN', { style: 'currency', currency: 'INR' })` for English
- Use `Intl.NumberFormat('hi-IN', { style: 'currency', currency: 'INR' })` for Hindi
- Example: ₹14,000.00 (English) / ₹14,000.00 (Hindi uses same numerals)

## Date Formatting
- English: 28 Mar 2026
- Hindi: 28 मार्च 2026
- Use `Intl.DateTimeFormat` with appropriate locale

## Product Translations
- Products have `name` + `name_hi` and `description` + `description_hi`
- Show appropriate version based on selected language
- Admin enters both translations when creating/editing products
